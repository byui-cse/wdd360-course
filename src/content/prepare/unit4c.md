## Reading

...Or if you would rather here is a list of short articles to read about the topics.

### 1. The Practical Test Pyramid

[The Practical Test Pyramid](https://martinfowler.com/articles/practical-test-pyramid.html) by Martin Fowler

This comprehensive guide explains the testing pyramid concept (unit → integration → E2E), when to use each type of test, and how to balance your test suite. Essential reading for understanding what kinds of tests to write and why.

### 2. Introduction to Test Driven Development (TDD)

[Test-Driven Development: By Example](https://tidyfirst.substack.com/p/canon-tdd) - Kent Beck's classic explanation of the Red-Green-Refactor cycle.

[What is Test Driven Development?](https://www.freecodecamp.org/news/test-driven-development-what-it-is-and-what-it-is-not-41fa6bca02a2/) - A beginner-friendly intro to TDD philosophy with clear examples from freeCodeCamp.

### 3. Write Tests. Not Too Many. Mostly Integration.

[Write Tests. Not Too Many. Mostly Integration.](https://kentcdodds.com/blog/write-tests) by Kent C. Dodds

An influential piece on balancing different test types and writing tests that give confidence without brittleness. Great guidance on what to test and what not to test.

### 4. Testing Best Practices (Playwright Documentation)

[Playwright Best Practices](https://playwright.dev/docs/best-practices)

Official Playwright guide covering locator strategies, test isolation, and avoiding flaky tests. Essential for writing reliable integration tests.

### 5. Vitest Guide

[Vitest - Why Vitest?](https://vitest.dev/guide/why.html)

Introduction to Vitest's philosophy and features. Helpful context for the unit testing tool we'll be using in the activity.
