## Overview

In this prove activity you will refactor a small front-end app from direct DOM manipulation to reusable custom web components.

Assume you have completed [Ponder: Custom Web Components](../../ponder/components/). You should already be comfortable with:

- Creating custom elements with `HTMLElement`.
- Using lifecycle methods (`constructor`, `connectedCallback`, etc.).
- Passing data with attributes/properties.
- Styling with Shadow DOM.

## Instructions

Build the starter app first, verify it works, then refactor it to web components.

## Core Requirements

### **01** Build and Run the Starter App

Create the files below in a new folder and run them with Live Server (or any static local server).

`index.html`

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>User Directory</title>
    <link rel="stylesheet" href="styles.css" />
  </head>
  <body>
    <main>
      <h1>User Directory</h1>
      <button id="loadUsersBtn">Load Users</button>
      <p id="status"></p>
      <ul id="userList"></ul>
    </main>

    <script type="module" src="app.js"></script>
  </body>
</html>
```

`styles.css`

```css
body {
  font-family: Arial, sans-serif;
  margin: 2rem;
  background: #f8fafc;
  color: #1f2937;
}

h1 {
  margin-top: 0;
}

#loadUsersBtn {
  padding: 0.5rem 0.8rem;
  cursor: pointer;
}

#userList {
  list-style: none;
  padding: 0;
  margin-top: 1rem;
  display: grid;
  gap: 0.75rem;
}

#userList li {
  background: white;
  border: 1px solid #e5e7eb;
  border-radius: 8px;
  padding: 0.75rem;
}
```

`app.js`

```js
const button = document.getElementById('loadUsersBtn')
const statusEl = document.getElementById('status')
const userList = document.getElementById('userList')

button.addEventListener('click', loadUsers)

async function loadUsers() {
  statusEl.textContent = 'Loading users...'
  userList.innerHTML = ''

  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/users')
    if (!response.ok) throw new Error('Failed to fetch users')

    const users = await response.json()
    renderUsers(users)
    statusEl.textContent = `${users.length} users loaded.`
  } catch (error) {
    statusEl.textContent = `Error: ${error.message}`
  }
}

function renderUsers(users) {
  userList.innerHTML = users
    .map(
      (user) => `
      <li>
        <strong>${user.name}</strong><br />
        <span>${user.email}</span><br />
        <small>${user.company.name}</small>
      </li>
    `
    )
    .join('')
}
```

Run the app and confirm this behavior before refactoring:

1. Clicking the button fetches users from the API.
2. Status text updates during loading and after completion.
3. User cards render to the page.

### **02** Refactor to Web Components

Refactor the app so it uses **at least two custom elements**:

1. `user-directory` (container component)
2. `user-card` (single user display component)

Requirements:

1. Keep the same API endpoint (`https://jsonplaceholder.typicode.com/users`).
2. Move rendering logic into your components.
3. Use at least one attribute (or property) to pass data into `user-card`.
4. Add Shadow DOM styling in `user-card`.
5. Keep behavior the same: click button, fetch users, render list, show status.

<Details summary="Suggested architecture">

- `user-directory`
  - Owns button/status and fetch logic.
  - Creates one `user-card` per user.
  - Appends cards to a container.
- `user-card`
  - Receives user data.
  - Renders markup in shadow root.
  - Owns its own CSS.

</Details>
